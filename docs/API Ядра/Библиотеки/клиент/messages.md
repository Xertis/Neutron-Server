# Messages

>[!WARNING]
> Перед чтением документации к клиентской части **api.messages**, рекомендуется прочитать серверную часть

## Отправка сообщений

```lua
-- Отправляет данные на сервер
-- Message:send(pack: string, data: table)
PingMessage:send({ id = "abc", data = {1, 2, 3} })
```

`data` — таблица в виде, описанном схемой сообщения. Кодирование в байты и
вызов `events.send` происходят внутри метода — отдельно собирать буфер
не нужно.

## Обработка сообщений

```lua
-- Регистрирует обработчик входящих сообщений от сервера
-- Message:on(pack: string, handler: function(bytearray))
PingMessage:on(function(data)
    -- data уже распаршена согласно схеме сообщения
    print(data.id, #data.data)
end)
```

В `handler` приходят не байты, а уже декодированная таблица `data` —
весь разбор по схеме делает само сообщение перед вызовом обработчика.
