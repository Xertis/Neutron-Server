## Создание репликатора

`Replication` — обёртка над `api.messages`, которая берёт на себя
синхронизацию состояния таблиц между сервером и клиентом. Вместо того
чтобы вручную отправлять `Message:tell`/`Message:echo` при каждом изменении
данных, репликатор сам считает разницу между текущим и предыдущим
состоянием таблицы и отправляет только изменившиеся поля.

```lua
-- Replication.new(pack: string, event: string, schema: table) -> Replicator
local PosReplication = Replication.new("mypack", "pos_sync", {
    x = "float32",
    y = "float32",
    z = "float32"
})
```

`pack` — это имя пака, внутри которого репликатор.
`event` — имя ивента, который будет использоваться репликатором

>[!WARNING]
> `pack` и `event` **должны быть одинаковыми в коде сервера и клиента** для
одного и того же репликатора — именно по ним сопоставляются отправитель и
слушатель.

## Схема репликатора

Схема описывается так же, как схема обычного сообщения — таблица
`{ ключ = тип }` с поддержкой вложенных таблиц:

```lua
local EntityReplication = Replication.new("mypack", "entity_sync", {
    pos = { x = "float32", y = "float32", z = "float32" },
    hp = "uint16"
})
```

Порядок полей при кодировании фиксируется один раз при вызове
`Replication.new` одинаковым образом для сервера и клиента.

>[!WARNING]
> Схема репликатора **не может содержать тип `NullAble<T>`** — `Replication.new`
выбросит ошибку, если найдёт его на любом уровне вложенности. Это
ограничение связано с тем, что под капотом каждое поле схемы автоматически
оборачивается в `NullAble<T>` — так diff может передавать только
изменившиеся поля, оставляя остальные пустыми.

## Публичная реплика

```lua
-- Replicator:create_public_replica(id: int, initial_value: table, need_send: function(client, dirty): boolean?) -> table
local replica = PosReplicator:create_public_replica(1, {
    x = 0, y = 0, z = 0
})

-- изменение будет подхвачено автоматически и разослано всем
replica.x = 10.5
```

Публичная реплика по умолчанию рассылается всем подключённым клиентам

Если передан `need_send`, рассылка происходит не широковещательно, а
индивидуально для каждого клиента — функция вызывается с `client` и
изменившимися полями `dirty`, если она возвращает `true`, этому клиенту
отправляется новое состояние реплики.

```lua
local replica = PosReplication:create_public_replica(1, { x = 0, y = 0, z = 0 },
    function(client, dirty)
        return is_in_view_distance(client, replica)
    end)
```

## Приватная реплика

```lua
-- Replicator:create_private_replica(id: int, initial_value: table, client: Client) -> table
local inventory = InventoryReplication:create_private_replica(1, {
    slots = {}
}, client)
```

Приватная реплика всегда отправляется только одному, заранее заданному
`client` — аналог `Message:tell`, но с автоматическим расчётом diff.

## Удаление реплики

```lua
-- Replicator:remove_replica(id: int)
PosReplication:remove_replica(1)
```

Прекращает отслеживание изменений и удаляет реплику с указанным `id`.
Дальнейшие изменения возвращённой ранее таблицы никуда не отправляются.
