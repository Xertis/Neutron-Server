## Создание сообщения

`Message` — обёртка над `api.events`, которая берёт на себя кодирование и
декодирование данных сообщения по заданной схеме. Вместо ручной упаковки байт
в `events.tell`/`events.send`/`events.on` каждое сообщение описывает структуру
одного типа данных один раз, а дальше отправляет и принимает уже готовые
Lua-таблицы.

```lua
local Message = import "net/protocol/message"

-- Message.new(pack: string, event: string, schema: table) -> Message
local PingMessage = Message.new("mypack", "ping", {
    id = "string",
    data = "array<uint8>"
})
```

`pack` — это имя пака, внутри которого сообщение.
`event` — это строковый идентификатор сообщения внутри пака.

>[!WARNING]
> `pack` и `event` **должны быть одинаковыми в коде сервера и клиента** для одного и того же сообщения —
именно по ним `api.events` сопоставляет отправителя и обработчик.
## Схема сообщения

Схема — обычная таблица `{ ключ = тип }`. Порядок полей при кодировании
фиксируется один раз при вызове `Message.new` одинаковым образом для сервера и клиента

```lua
local MoveMessage = Message.new("move", {
    x = "float32",
    y = "float32",
    z = "float32"
})
```

Поля можно вкладывать друг в друга — значением ключа указывается вложенная
схема (таблица), а не строка-тип:

```lua
local SpawnMessage = Message.new("mypack", "spawn", {
    name = "string",
    pos = { x = "float32", y = "float32", z = "float32" }
})
```

При отправке/получении такого сообщения данные выглядят как обычная
вложенная таблица: `data.pos.x`, `data.name` и т.д.

## Доступные типы

Типы указываются в схеме как строки (`{ key = "тип" }`). Ниже — основные
типы, доступные для полей сообщения.

| Тип              | Размер     | Описание                                                                                                      |
| ---------------- | ---------- | ------------------------------------------------------------------------------------------------------------- |
| `boolean`        | 1 бит      | Логическое значение                                                                                           |
| `uint8`          | 1 байт     | Целое без знака, 0..255                                                                                       |
| `int8`           | 1 байт     | Целое со знаком, -127..127                                                                                    |
| `uint16`         | 2 байта    | Целое без знака, 0..65535                                                                                     |
| `int16`          | 2 байта    | Целое со знаком                                                                                               |
| `uint24`         | 3 байта    | Целое без знака, 0..16777215                                                                                  |
| `int32`          | 4 байта    | Целое со знаком                                                                                               |
| `uint32`         | 4 байта    | Целое без знака                                                                                               |
| `int64`          | 8 байт     | Целое со знаком                                                                                               |
| `float32`            | 4 байта    | Число с плавающей точкой одинарной точности                                                                   |
| `float64`            | 8 байт     | Число с плавающей точкой двойной точности                                                                     |
| `var` | переменный | Целое без знака, компактная кодировка до 8 байт |
| `hex`            | 4 бита     | Одна hex-цифра (0-15)                                                                                         |
| `norm8`          | 1 байт     | Нормализованное значение -1.0..1.0, упакованное в байт                                                         |
| `string`         | переменный | Строка переменной длины                                                                                       |
| `bytearray`      | переменный | Произвольный массив байт переменной длины                                                                     |                                     |
| `bson`           | переменный | Произвольная структура, сериализованная как BSON                                                              |

### Generic-обёртки

Эти типы принимают вложенный тип элемента и не самостоятельны — используются
как обёртка вокруг другого типа:

| Тип           | Описание                                                                        |
| ------------- | ------------------------------------------------------------------------------- |
| `Array<T>`    | Список элементов типа `T` переменной длины (например, `Array<uint8>`)           |
| `NullAble<T>` | Опциональное значение типа `T` (1 бит флага наличия + само значение, если есть) |
| `Vec2<T>`     | Вектор из двух значений типа `T`                                                |
| `Vec3<T>`     | Вектор из трёх значений типа `T`                                                |

Так же спокойно допускается подобная конструкция вида `Vec2<Vec3<Vec2<uint8>>>`

## Отправка сообщений

Message умеет отправлять данные клиентам — точечно или всем подключённым:

```lua
-- Отправляет данные указанному клиенту
-- Message:tell(pack: string, client: Client, data: table)
PingMessage:tell(client, { id = "abc", data = {1, 2, 3} })

-- Отправляет данные всем подключённым клиентам
-- Message:echo(pack: string, data: table)
PingMessage:echo({ id = "abc", data = {1, 2, 3} })
```

`data` — таблица в виде, описанном схемой сообщения. Кодирование в байты и
вызов `events.tell`/`events.echo` происходят внутри метода — отдельно
собирать буфер не нужно.

## Обработка сообщений

```lua
-- Регистрирует обработчик входящих сообщений от клиента
-- Message:on(pack: string, handler: function(Client, table))
PingMessage:on(function(client, data)
    -- data уже распаршена согласно схеме сообщения
    print(data.id, #data.data)
end)
```

В `handler` приходят не байты, а уже декодированная таблица `data` —
весь разбор по схеме делает само сообщение перед вызовом обработчика.
